# External API Services

youtui talks to several external APIs for music playback, metadata, lyrics,
and scrobbling. Most work with zero config. Some need tokens.

## Quick Start — Full Config Template

Everything goes in `~/.config/youtui/config.toml`:

```toml
[auth]
auth_type = "browser"          # "browser"(default) | "oauth" | "none"
cookie_browser = "chromium"    # chromimum|firefox|brave|safari|etc

[scrobbling]
enabled = false                # set true to scrobble to Last.fm
api_key = ""                   # Last.fm API key (metadata + scrobbling)
api_secret = ""                # Last.fm secret (scrobbling only)
session_key = ""               # Last.fm session (scrobbling only)
genius_token = ""              # Genius access token (lyrics + annotations)
discogs_token = ""             # Discogs personal access token

[audio]
downloader_type = "yt-dlp"    # "yt-dlp"(default) | "native"
```

Only `auth_type` is required. Everything else is optional — missing tokens
just skip that provider silently.

## All Tokens and Keys Reference

| Token | Where to Get | Config Field | What Breaks Without It |
|-------|-------------|--------------|------------------------|
| YTM auth | auto (browser cookies / OAuth) | `auth_type` | Everything — app won't start |
| Last.fm API key | https://www.last.fm/api/account/create | `api_key` | Metadata from Last.fm, album art, scrobbling |
| Last.fm secret | same page as API key | `api_secret` | Scrobbling (signing required) |
| Last.fm session key | generated by youtui on first scrobble | `session_key` | Scrobbling |
| Discogs token | https://www.discogs.com/settings/developers | `discogs_token` | Discogs metadata (tracklists, genres) |
| Genius token | https://genius.com/api-clients | `genius_token` | Annotations (lyrics work without it) |
| `MA_COOKIE` | browser DevTools on metal-archives.com | env var | Metal Archives metadata |

## Per-Platform Setup

### YouTube Music Auth

YTM is required. One of these must work:

**Browser cookies (default)** — reads from your browser's cookie file:

| Browser | Linux | macOS | BSD |
|---------|-------|-------|-----|
| Chromium | `~/.config/chromium/` | `~/Library/Application Support/Chromium/` | `~/.config/chromium/` |
| Chrome | `~/.config/google-chrome/` | `~/Library/Application Support/Google/Chrome/` | `~/.config/google-chrome/` |
| Firefox | `~/.mozilla/firefox/` | `~/Library/Application Support/Firefox/` | `~/.mozilla/firefox/` |
| Brave | `~/.config/BraveSoftware/Brave-Browser/` | `~/Library/Application Support/BraveSoftware/Brave-Browser/` | `~/.config/BraveSoftware/` |
| Safari | — | `~/Library/Cookies/` | — |

Set `cookie_browser` in config to match your browser. Default is `"chromium"`.

**OAuth** — interactive device-code flow. Set `auth_type = "oauth"` in config.
Runs on first launch. Token saved to `~/.local/share/youtui/`.

**No auth** — set `auth_type = "none"`. Read-only mode: no library, no
playlists, no ratings. Search and browse only.

### Last.fm

1. Register at https://www.last.fm/api/account/create
2. Copy the **API key** and **shared secret**
3. Add to config:
   ```toml
   [scrobbling]
   api_key = "abc123"
   api_secret = "xyz789"
   ```
4. For metadata-only (no scrobbling), just `api_key` is enough
5. For scrobbling, also set:
   ```toml
   enabled = true
   ```
   Session key is auto-generated on first scrobble — you do not need to set it.

**Platform notes**: Last.fm API works identically on Linux, macOS, BSD.
Rate limit: 5 req/sec (generous — youtui stays well under).

### Discogs

1. Go to https://www.discogs.com/settings/developers
2. Generate a **Personal Access Token**
3. Add to config:
   ```toml
   [scrobbling]
   discogs_token = "your_token_here"
   ```

**Platform notes**: Same token file works everywhere. Rate limit: 1 req/sec
authenticated, 25 req/min unauthenticated. Token raises limit to 1/sec.

### Genius

1. Register at https://genius.com/api-clients
2. Create a new app, generate **Client Access Token**
3. Add to config:
   ```toml
   [scrobbling]
   genius_token = "your_token_here"
   ```

**Without token**: Lyrics still work (you can use LRCLIB or the page-scrape
fallback). Annotations (explanations for specific lines) need the token.

**Platform notes**: Genius token works everywhere. Rate limit: 40 req/min
unauthenticated, 200 req/min with token.

### MusicBrainz

No setup needed. Free and open API. Runs automatically if reachable.

**Platform notes**: Rate limit: 1 req/sec. youtui serializes requests
(Semaphore 1) to stay under this. Works on all platforms.

### Metal Archives

Requires a Cloudflare bypass cookie. The cookie expires ~30 min.

1. Open https://www.metal-archives.com/ in your browser
2. DevTools → Application → Cookies → copy `cf_clearance` value
3. Export as environment variable:
   ```bash
   # Linux/macOS/BSD (bash/zsh):
   export MA_COOKIE="cf_clearance=your_value"
   youtui

   # Or add to ~/.bashrc / ~/.zshrc / ~/.profile
   ```

**Per-platform persistence**:

| Platform | File to add `export MA_COOKIE=...` |
|----------|-----------------------------------|
| Linux (bash) | `~/.bashrc` |
| Linux (zsh) | `~/.zshrc` |
| macOS (zsh) | `~/.zprofile` |
| BSD (sh) | `~/.profile` |

Without `MA_COOKIE`, Metal Archives is skipped. Other providers still run.

### LRCLIB

Free synchronized lyrics API. No setup, no token. Works everywhere.

## Metadata Pipeline Order

Providers run in priority order (lower = checked first). First hit wins.

| Priority | Provider | Requires |
|----------|----------|----------|
| 5 | Metal Archives | `MA_COOKIE` env var |
| 7 | MusicBrainz | nothing |
| 8 | Discogs | `discogs_token` in config |
| 10 | Last.fm album | `api_key` in config |
| 15 | YTM album enrichment | (uses YTM client auth) |
| 20 | Last.fm track | `api_key` in config |
| 40 | Genius | `genius_token` in config |

## Lyrics Pipeline Order

| Priority | Provider | Requires |
|----------|----------|----------|
| 1 | Genius (slug URL + API) | nothing (token optional for annotations) |
| 2 | LRCLIB | nothing |
| 3 | bandcamp-lyrics CLI | external `bandcamp-lyrics` binary |
| 4 | lyr CLI | external `lyr` binary |

## Cache

Metadata results are cached at `~/.local/share/youtui/metadata_cache.json`.
200-entry LRU. Survives restarts. Clear by deleting the file.

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| "Auth failed" on startup | No browser cookies found | Run browser once, or use `auth_type = "oauth"` |
| No album art in footer | Missing Last.fm API key | Add `api_key` to config |
| No lyrics | No Genius token, LRCLIB unreachable | Add `genius_token`, or check network |
| Scrobble not working | `enabled = false` or missing `api_secret` | Set `enabled = true` and add secret |
| Metal Archives not returning data | `MA_COOKIE` missing or expired | Refresh cookie in browser |
| Metadata cache stale | Old cached entries | Delete `metadata_cache.json` |
| Album art broken in tmux | Sixel passthrough disabled | See [08-known-issues.md](08-known-issues.md) |
