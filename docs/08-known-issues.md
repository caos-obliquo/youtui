# Known Issues

## Tmux + Sixel Album Art

| Issue | Workaround |
|-------|------------|
| **Footer album art broken in tmux**: Sixel rendering corrupts when running inside tmux. Footer shows garbage or missing image. | `:reload` re-renders everything, often fixes it. Also try `tmux set -g allow-passthrough on` in `~/.tmux.conf`. |
| **`o.v` album art popup broken in tmux**: Same root cause. Sixel overlay can render partially, fail to clear, or leave artifacts. | Same fix. If popup persists after close, `:reload` clears it fully. |
| **No sixel support at all**: Terminal emulator does not support sixel graphics. | Album art silently skipped. Check with `printf '\eP#? \e\\'` — no response = no sixel. |

### What is sixel?

Sixel is a DEC terminal protocol for displaying bitmap graphics inline. It sends raw pixel data as escape sequences. Not all terminals support it. Youtui uses sixel for:
- Footer album art (small thumbnail next to progress bar)
- `o.v` full-size album art popup

### Which terminals support sixel?

| Terminal | Platform | Sixel |
|----------|----------|-------|
| **foot** | Linux (Wayland) | ✅ Native |
| **kitty** | Linux, macOS, BSD | ✅ (icat kitten, not native sixel) |
| **Alacritty** | Linux, macOS, BSD | ❌ No |
| **GNOME Terminal** | Linux | ❌ No |
| **iTerm2** | macOS | ✅ (set `terminal-enable-sixel` = yes) |
| **WezTerm** | Linux, macOS, BSD | ✅ |
| **XTerm** | Linux, BSD | ✅ (build with `--enable-sixel`) |
| **mlterm** | Linux, BSD | ✅ |
| **Windows Terminal** | Windows | ❌ No |
| **Konsole** | Linux | ❌ No |

### tmux specifics per platform

tmux intercepts terminal output and re-encodes it. Sixel passthrough must be enabled:

**Foot + tmux on Linux/BSD**:
```tmux
set -g -a terminal-overrides ',foot*:Ms=\E]52;%p1%s;%p2%s\7'
set -g allow-passthrough on
```

**iTerm2 + tmux on macOS**:
```tmux
set -g -a terminal-overrides ',iterm2*:Ms=\E]52;%p1%s;%p2%s\7'
set -g allow-passthrough on
```

**XTerm + tmux on Linux/BSD**:
```tmux
set -g -a terminal-overrides ',xterm*:Ms=\E]52;%p1%s;%p2%s\7'
set -g allow-passthrough on
```

**Kitty + tmux**:
```tmux
set -g -a terminal-overrides ',xterm-kitty:Ms=\E]52;%p1%s;%p2%s\7'
set -g allow-passthrough on
```

The `terminal-overrides` line tells tmux to pass through the sixel escape sequence for your specific terminal type. Without it, tmux blocks all escape sequences it does not recognize.

### Why `:reload` fixes it

`:reload` triggers a full UI re-render:
1. Clears the entire screen (including any corrupted sixel state)
2. Re-draws all widgets from scratch
3. Re-displays the footer with fresh album art

This bypasses any partial corruption in sixel state tracking that accumulated during tmux session. It is the universal "fix everything" command for display glitches.
